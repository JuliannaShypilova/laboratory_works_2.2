# Задача 31.4: Власна черга з пріоритетами на базі deque

Програма реалізує механізм, де пріоритетні повідомлення обробляються раніше за звичайні, навіть якщо вони надійшли пізніше.

## Особливості реалізації:
* **Клас `PriorityDeque`:** створений на базі стандартного `collections.deque`.
* **Логіка пріоритетів:** * Повідомлення з високим пріоритетом додаються в початок черги за допомогою `appendleft()`.
  * Звичайні повідомлення додаються в кінець через `append()`.
* **Синхронізація:** використано `threading.Lock()` для забезпечення безпечної роботи з чергою з декількох потоків (thread-safety).

## Як це працює:
Генератор швидко наповнює чергу завданнями. Обробник починає роботу з невеликою затримкою, що дозволяє пріоритетним завданням опинитися попереду списку на виконання.
